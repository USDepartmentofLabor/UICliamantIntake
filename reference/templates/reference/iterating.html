All state workforce agencies (SWAs) are working in challenging environments that include legacy technology. States have limited resources and a workload that can be unpredictable. These factors can make large scale modernization difficult, expensive, and/or time-consuming. However, there are opportunities to make incremental and impactful changes.

The case study below demonstrates how the state of New Jersey has successfully iterated despite their current technical challenges.

Go mobile-friendly with existing technology

Mobile-friendly systems and websites are critical given that smart phones are the most accessible (and often only device) for many applicants. Currently, few states have mobile-friendly experiences today.

Using CSS media queries, you can make huge improvements to the mobile-friendliness of your site, often without the need to upgrade your software stack or infrastructure. Have a conversation with your engineering experts and share this code to see what’s possible in the short term.

Media query example

The following code snippet shows a couple sections of text aligned side-by-side on a page using the CSS flex display property. This works well for wide screens, but appears too narrow on mobile. 

<style>
  .container {
    display: flex;
    gap: 1rem;
  }
  .container div {
    background-color: #eee;
    padding: 1rem;
    flex-grow: 1;
  }
</style>

  </div>

To fix this, we can add a media query. The added code in the following snippet translates to "apply these styles if the viewport width is less than or equal to 480 pixels." In this case, we change the container from display: flex to display: block and add some margin around the container's children. 

<style>
  .container {
    display: flex;
    gap: 1rem;
  }
  .container div {
    background-color: #eee;
    padding: 1rem;
    flex-grow: 1;
  }
  /* Added media query */
  @media (max-width: 480px) {
    .container {
      display: block;
    }
    .container div {
      margin: 1rem;
    }
  }
</style>
<div class="container">
  <div>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod
    tempor incididunt ut labore et dolore magna aliqua.
  </div>
  <div>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod
    tempor incididunt ut labore et dolore magna aliqua.
  </div>
</div>

With this change in hand, we can adjust our screen size and see that, when we cross the 480px breakpoint, our layout changes to be more mobile-friendly.

Building outside your existing systems

When you're not constrained by legacy systems and process, you can make headway fast and may uncover ideas that you otherwise wouldn't have.

Experiment from scratch

USDS and USDOL build a pilot application with New Jersey on a brand new site entirely outside of their existing website and application. New Jersey is not releasing this pilot app as their new system, but they are taking pieces of it to incorporate into their existing system for improvement. You can try building out a specific interaction on the side, you may eventually incorporate it into your overall system, or you may just learn new things that inform your system’s roadmap.

Decouple content from code

Decoupling content updates from shipping code may help you deliver content changes faster to production. This can be especially helpful if pushing code changes to production is challenging or finicky or if you simply want to have different personnel/processes that manage content than write code.

If this sounds beneficial, consider looking into commercial software-as-a-service products under the headless content management system (CMS) category. Typically, these products give you an intuitive, no-code user interface in which you can manage the content that appears in your application. Your developers can use the tool's Application Programming Interface (API) to dynamically pull the content into the application. Your application will therefore always be up-to-date with the content management tool you use and you won't have to redeploy your application to make simple content updates.

Make space to hear ideas

New Jersey has struggled with legacy technology which has stiffled innovation for years. The state found that breaking down silos between policy and technology experts created space for new ideas and opportunities.

When UI leadership in New Jersey came together with the IT team to brainstorm, they were able to unlock ideas for creative, near-term workarounds to make incremental plain language and mobile friendly changes to their online UI application. Now New Jersey is able to push forward some important adjustments to the experience today, while working on the longer term tech modernization goals in parallel. How to run an effective meeting

  • Have a stated purpose, desired outcome, agenda, and format.
  • Make sure participants know what is expected of them, and what types of participation would be welcome. For example, if you want to hear innovative new ideas from junior staff members, make sure they know that they do not need approval from their chain of command to present an idea.
  • The format and paticipants should match the desired outcome. For example, if >lthe meeting needs to result in an important decision, the person with authority to make that decision must be present.
  • Stay true to the goals of the meeting; if the discussion goes off-track, offer to schedule follow-up discussions on those topics.
  • Next steps should be documented with owners and due dates clearly identified.
  • Be preprared to schedule follow-up sessions to investigate promising ideas and dive deeper into the risks and level of effort involved in top ideas.
  • Speak when you believe it will add value; avoid repeating what others have said.
