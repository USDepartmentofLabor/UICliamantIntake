All state workforce agencies (SWAs) are working in challenging environments that include legacy technology. States have limited resources and a workload that can be unpredictable. These factors can make large scale modernization difficult, expensive, and/or time-consuming. However, there are opportunities to make incremental and impactful changes.

The case study below demonstrates how the state of New Jersey has successfully made significant improvements despite their current technical challenges.

Go mobile-friendly with existing technology

Mobile-friendly systems and websites are critical given that smart phones are the most accessible (and often only) for many applicants. Few states have mobile-friendly experiences today. Around 60% of states and territories do not have responsive sites for unemployment benefits. Additionally, multiple studies show that 60-80% of applicants have smartphones as their primary device.

Using CSS media queries, you can make huge improvements to the mobile-friendliness of your site, often without the need to upgrade your software stack or infrastructure. Have a conversation with your engineering experts and share this code to see what's possible in the short term.

Media query example

The following code snippet shows a couple sections of text aligned side-by-side on a page using the CSS flex display property. This works well for wide screens, but appears too narrow on mobile. 

<style>
  .container {
    display: flex;
    gap: 1rem;
  }
  .container div {
    background-color: #eee;
    padding: 1rem;
    flex-grow: 1;
  }
</style>
<div class="container">
  <div>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod
    tempor incididunt ut labore et dolore magna aliqua.
  </div>
  <div>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod
    tempor incididunt ut labore et dolore magna aliqua.
  </div>
</div>

To fix this, we can add a media query. The added code in the following snippet translates to "apply these styles if the viewport width is less than or equal to 480 pixels." In this case, we change the container from display: flex to display: block and add some margin around the container's children. 

<style>
  .container {
    display: flex;
    gap: 1rem;
  }
  .container div {
    background-color: #eee;
    padding: 1rem;
    flex-grow: 1;
  }
  /* Added media query */
  @media (max-width: 480px) {
    .container {
      display: block;
    }
    .container div {
      margin: 1rem;
    }
  }
</style>
<div class="container">
  <div>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod
    tempor incididunt ut labore et dolore magna aliqua.
  </div>
  <div>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod
    tempor incididunt ut labore et dolore magna aliqua.
  </div>
</div>

With this change in hand, we can adjust our screen size and see that, when we cross the 480px breakpoint, our layout changes to be more mobile-friendly. For the example below, the mobile version automatically vertically stacks content to be mobile-friendly.

Experiment from scratch

When you are not constrained by legacy systems and process, you can make fast progress with new capabilities. You may uncover ideas that you otherwise wouldn't have.

In partnership with USDOL, USDS built a pilot application with New Jersey on a separate, new site entirely outside of their existing website and application. New Jersey is not releasing this application as their new system, but are using pieces of it to improve their existing system. If you build on the side, you may find valuable features to incorporate into your system(s) or you may learn new details that inform your technology roadmap.

Change the frontend without requiring changes on the backend

A challenge to making changes to application form questions is that it could require changes to the database (or backend). But, it may be possible to update the applicant-facing frontend of the application without changing the backend. This is done by mapping new field options to existing options.

By changing the frontend form responses and developing a mapping schema, you can still conform to the database requirements. This enables changes on form questions to be made without a significant tech overhaul.

Make space to hear ideas

New Jersey, like most states and territories, has legacy technology that has stifled innovation for years. They found that breaking down silos between policy and technology experts created space for new ideas and opportunities.

When UI Program Office in New Jersey met with the information technology (IT) staff to brainstorm, they generated creative ideas to make incremental plain language and mobile-friendly changes to the UI application. They were able to make important adjustments to the claimant experience today while working on the longer-term tech modernization goals in parallel.

New Jersey, like many states and territories, has struggled to update their UI application because of complicated legacy technology problems. During the pilot, they brought together policy, IT, and innovation teammates to meet and ideate. They came up with technical workarounds that enabled them to make short term improvements.

Below are some of the tactics New Jersey used to create space for these ideas and act on them. If you're inspired, and haven't already, try this in your organization to see if it results in any new learnings and opportunities.

NJ structured the meeting around an agenda like this:

  • What are the program office's goals? Define specific claimant-experience goals such as plain language for application questions, accessibility, or mobile-responsiveness.
  • What are the blockers? Whether it's a prioritization, technology, or other problem, discuss the blockers in detail.
  • What are some ideas for workarounds? Discuss how to solve the problems or change incrementally to reach claimant experience goals?

New Jersey made sure IT, Innovation, and Policy leadership were present at the meeting. Decision-makers from those areas were present to advocate for human centered design and to hear the various ideas. After leaders decided which ideas to pursue, they scheduled follow-up sessions to dive into implementation details so that tactical logistics did not interfere with the ideation meeting.

New Jersey used a round-table style to facilitate the meeting so leaders as well anyone from the technology side had the opportunity to speak. It specifically was important that the technical experts had the space to propose solutions based on what policy wanted to achieve.
